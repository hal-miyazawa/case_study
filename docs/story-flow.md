# Story Flow

このファイルは、AI を使った防災対策ゲーム版のストーリー全体の流れと、画面ごとの内容メモです。

今回の方針では、固定された一本道のシナリオではなく、プレイヤーが選んだ行動やショップで買ったものをもとに、AI が次の会話や状況説明を生成して進行します。

## 全体のコンセプト

主人公はテレビで「数日後に大きな地震が起こる」という占い師の発言を聞く。

最初は半信半疑だが、テレビの言葉や自分の部屋の危険な状態を見て、不安を覚える。プレイヤーは主人公として、地震が起こる前にできるだけ多くの対策を行う。

対策フェーズでは、ショップに行って防災グッズを買ったり、家族や周りの人に連絡したり、部屋の中を確認したりできる。行動は画面内のアイコンやボタンをタップして選ぶ。

地震発生後は、それまでに選んだ行動と購入したアイテムをもとに展開が変わる。AI はその状態に合わせて、自然な会話文や状況説明、次の選択肢を返す。

## 基本の流れ

1. テレビで占い師の発言を見る
2. 主人公が地震への不安を感じる
3. 対策フェーズに入る
4. プレイヤーが自由行動を選ぶ
5. ショップで防災グッズを購入できる
6. 家族や周囲に連絡できる
7. 部屋の中でできる対策を行う
8. 選んだ行動や持ち物を AI に渡す
9. AI が次の状況説明や選択肢を生成する
10. 一定ターン後、または条件達成後に地震が発生する
11. 準備内容をもとにエンディングへ進む

## ゲームの進め方

今回のゲームは、固定の選択肢だけで進むのではなく、次のような状態を持って進行する。

- 残り時間
- 所持金
- 購入したアイテム
- 家で実行した対策
- 家族や周囲に連絡したか
- 地震発生後に必要な備えがあるか

プレイヤーが行動を選ぶたびに、これらの状態が変わる。

AI には、現在の状態とプレイヤーが選んだ行動を渡す。AI はそれをもとに、次のセリフ、状況説明、選択肢を返す。

## 画面ごとの内容

### 1. テレビを見る画面

目的:

- 地震が起こるかもしれないという導入を作る
- 主人公が行動を始める理由を作る
- 占い師の言葉によって、少し不気味な予兆を出す

内容:

- 主人公が自宅でテレビを見ている
- 番組内で占い師が「数日後に大きな地震が起こる」と話す
- 周囲の出演者は半分冗談のように受け流す
- 主人公だけが妙にその言葉を気にする

表示例:

```text
占い師:
「数日後、この街で大きな揺れが起こります。備えのない家ほど、危険です。」

主人公:
「地震……？ ただの占いだよな。でも、なんだか引っかかる。」
```

遷移:

- 次に `対策開始画面` へ進む

### 2. 対策開始画面

目的:

- プレイヤーに自由行動が始まることを伝える
- 何をすればよいかを考えさせる
- 部屋や家族、ショップなどの行動先を示す

内容:

- 主人公が部屋を見回す
- 棚、窓、テレビ、スマホ、玄関などのアイコンを表示する
- アイコンをタップすると、それぞれの行動に進む

選べる行動例:

- ショップへ行く
- 家族に連絡する
- 近所の人に声をかける
- 部屋の危険な場所を確認する
- 避難場所を確認する
- 何もせず時間を進める

### 3. 自由行動フェーズ

目的:

- プレイヤー自身に防災対策を選ばせる
- 選んだ行動によって後の展開を変える
- AI が状況に合う会話や次の選択肢を生成できるようにする

内容:

- 画面上のアイコンやボタンから行動を選ぶ
- 1回行動すると時間が進む
- 行動によって持ち物やフラグが変化する
- AI がその行動の結果を文章で返す

行動例:

```text
ショップに行く
家族に連絡する
近所に伝える
部屋を片付ける
避難場所を調べる
非常用バッグをまとめる
```

AI に渡す情報の例:

```text
現在の状態:
- 残り時間: 3ターン
- 所持金: 5000円
- 購入済み: 飲料水、懐中電灯
- 実行済み: 家族に連絡した
- 今選んだ行動: 部屋の危険な場所を確認する

この状態に合わせて、次の状況説明と選択肢を返してください。
```

### 4. ショップ画面

目的:

- 防災グッズを購入できるようにする
- 買ったものによって、地震発生後の展開を変える
- 現実的な防災知識にもつながるようにする

内容:

- ショップには3つのカテゴリがある
- 各カテゴリに6つずつ、合計18個の商品を置く
- 所持金や持てる数に制限をつけると、選択に意味が出る

カテゴリ:

```text
安全対策
食料
医療系
```

## ショップの商品

### 安全対策

| 商品 | ゲーム内での分岐 |
| --- | --- |
| 家具固定器具 | 棚や家具が倒れるかどうかに関係する |
| 窓ガラス飛散防止フィルム | ガラスでけがをするか、避難経路が安全かに関係する |
| 懐中電灯 | 停電後に部屋を移動できるかに関係する |
| 携帯ラジオ | スマホが使えない時に避難情報を得られるかに関係する |
| モバイルバッテリー | 家族連絡や情報確認を続けられるかに関係する |
| 軍手・厚底スリッパ | 割れたガラスや散乱物の上を安全に歩けるかに関係する |

### 食料

| 商品 | ゲーム内での分岐 |
| --- | --- |
| 飲料水 | 地震後の避難生活で水分を確保できるかに関係する |
| 非常食セット | 避難後に体力を維持できるかに関係する |
| 缶詰 | 火や水が使えない状況でも食べられる |
| 栄養補助食品 | すぐ持ち出せる食料として使える |
| レトルトご飯・保存食 | 自宅に残る場合の備蓄として使える |
| 簡易トイレ | 断水時や避難生活の快適さに関係する |

### 医療系

| 商品 | ゲーム内での分岐 |
| --- | --- |
| 救急セット | 軽いけがを処置できるかに関係する |
| 常備薬 | 持病や体調不良イベントに対応できるかに関係する |
| 消毒液・ウェットシート | 傷や衛生状態の悪化を防げるかに関係する |
| マスク | 避難所、粉じん、衛生面のイベントに関係する |
| 体温計 | 体調管理や避難所での判断に関係する |
| 冷却シート・保温シート | 暑さ、寒さ、避難所生活の分岐に関係する |

## 分岐に使う重要アイテム

18個すべてを細かく判定すると複雑になるため、最初は特に重要なアイテムを中心に分岐させる。

重要アイテム:

```text
家具固定器具
飲料水
非常食セット
懐中電灯
モバイルバッテリー
救急セット
```

この6つはエンディング判定に直接関係させる。

その他のアイテムは、AI の文章や細かいイベントに反映する。

例:

- 窓ガラス飛散防止フィルムがあると、ガラス破片によるけがを避けやすい
- 携帯ラジオがあると、停電後でも避難情報を得やすい
- 簡易トイレがあると、避難生活のストレスが減る
- 常備薬があると、体調不良イベントに対応できる

## AI に任せること

AI は、プレイヤーの行動や持ち物に合わせて、自然な会話文や状況説明を作る。

AI に任せる内容:

- 主人公のセリフ
- 家族や周囲の反応
- アイテムを使った時の描写
- 地震発生後の状況説明
- 次に選べる行動の候補
- エンディング前の演出文

AI に完全には任せない内容:

- 所持金の計算
- 残り時間の計算
- どのアイテムを買ったかの記録
- エンディングの最終判定
- 重要フラグの管理

ゲームのルールはアプリ側で管理し、AI は文章を作る係として使う。

## AI への入力イメージ

AI に送る情報は、毎回同じ形にすると実装しやすい。

```ts
type GameState = {
  timeLeft: number
  money: number
  purchasedItems: string[]
  contactedFamily: boolean
  warnedNeighbors: boolean
  checkedShelter: boolean
  fixedFurniture: boolean
  packedEmergencyBag: boolean
}
```

AI に渡す文章の例:

```text
あなたは防災ゲームのシナリオを作るAIです。

現在の状態:
- 地震発生まで残り2ターン
- 所持金は1200円
- 購入済みアイテム: 飲料水、非常食セット、懐中電灯
- 家族には連絡済み
- 近所にはまだ伝えていない
- 家具固定はまだしていない
- 避難場所は確認済み

プレイヤーが選んだ行動:
- ショップから家に戻る

条件:
- 購入済みアイテムを会話や描写に反映する
- 未対策の危険も少し示す
- 次に選べる行動を3つ返す
```

AI から返してほしい形式の例:

```json
{
  "speaker": "主人公",
  "message": "買ってきた水と非常食をリュックに詰めた。少し安心したけれど、部屋の棚がまだ固定されていないことが気にかかる。",
  "choices": [
    "家具を固定する",
    "近所の人に声をかける",
    "スマホで避難情報を確認する"
  ]
}
```

## 地震発生イベント

一定ターンが経過すると、地震が発生する。

地震発生時には、プレイヤーの準備状況をもとに展開を変える。

判定に使う要素:

- 家具を固定しているか
- 懐中電灯を持っているか
- 水や食料を持っているか
- 救急セットを持っているか
- 家族に連絡しているか
- 避難場所を確認しているか
- モバイルバッテリーを持っているか

イベント例:

```text
家具固定あり:
大きな棚は揺れたが、壁に固定されていて倒れなかった。

家具固定なし:
背の高い棚が大きく傾き、床に物が散乱する。逃げ道がふさがれかける。

懐中電灯あり:
停電しても足元を照らして移動できる。

懐中電灯なし:
部屋が真っ暗になり、割れた物を踏みそうになる。
```

## エンディング分岐

エンディングは、AI が文章を作るが、最終判定はアプリ側で行う。

### Bad End

条件例:

- 家具固定をしていない
- 懐中電灯がない
- 救急セットがない
- 家族や周囲に連絡していない
- 避難場所を確認していない

内容:

- 地震発生後に対応が遅れる
- けがや孤立など、準備不足の結果が出る
- 何が足りなかったのかを振り返れるようにする

### Normal End

条件例:

- いくつかの重要アイテムは持っている
- ただし、すべての対策はできていない
- 命は助かるが、不便や不安が残る

内容:

- 最低限の備えで助かる
- しかし、水や連絡手段、避難情報などに不足が出る
- 次はもっと準備しようと思わせる

### Good End

条件例:

- 家具固定をしている
- 水と非常食を持っている
- 懐中電灯かモバイルバッテリーを持っている
- 家族に連絡している
- 避難場所を確認している

内容:

- 地震発生後も落ち着いて行動できる
- 家族や周囲との連携もできる
- 防災対策が現実的に役立ったことを示す

### True End

条件例:

- 重要アイテムを十分に準備している
- 家族と周囲の両方に連絡している
- 避難場所を確認している
- 部屋の危険箇所を対策している
- AI の会話内で、他人を助ける行動も選んでいる

内容:

- 主人公だけでなく、家族や近所の人も助かる
- 防災は自分だけではなく、周りと共有することが大切だと伝える
- 最後に現実のプレイヤーへ、自分の部屋や備蓄を確認する問いかけを入れる

## 最小版の判定ロジック

最初の実装では、複雑にしすぎず、重要なフラグだけで判定する。

```ts
type StoryFlags = {
  hasFurnitureStopper: boolean
  fixedFurniture: boolean
  hasWater: boolean
  hasFood: boolean
  hasFlashlight: boolean
  hasBattery: boolean
  hasFirstAidKit: boolean
  contactedFamily: boolean
  warnedNeighbors: boolean
  checkedShelter: boolean
}
```

判定イメージ:

```ts
const safetyScore =
  Number(hasFurnitureStopper && fixedFurniture) +
  Number(hasWater) +
  Number(hasFood) +
  Number(hasFlashlight || hasBattery) +
  Number(hasFirstAidKit) +
  Number(contactedFamily) +
  Number(warnedNeighbors) +
  Number(checkedShelter)

if (!fixedFurniture) {
  ending = 'bad_unfixed_furniture'
} else if (safetyScore >= 7) {
  ending = 'true'
} else if (safetyScore >= 5) {
  ending = 'good'
} else if (safetyScore >= 3) {
  ending = 'normal'
} else {
  ending = 'bad'
}
```

## 実装時の注意

- AI にすべての分岐判断を任せない
- アプリ側で購入アイテム、残り時間、所持金、重要フラグを管理する
- AI には現在の状態を渡して、文章と次の選択肢を作らせる
- ショップの商品は18個あるが、最初の判定は重要アイテム中心にする
- 画像は事前にAI生成して、`src/assets` に入れて使う想定
- 画面内のアイコンをタップして行動を選べるようにする
- 同じアイテムを買っても、使わなければ効果が出ないものを作る
- 例: 家具固定器具は、買うだけでなく家で使う必要がある
- エンディング後は、どの備えが足りなかったかを表示できると学習につながる
