# Story Flow

このファイルは、最小完成版で想定しているストーリー全体の流れと、画面ごとの内容メモです。

画面名や遷移は Figma の画面名・プロトタイプ矢印をもとに整理する想定です。実装前に Figma 側の画面名、矢印、分岐条件と照らし合わせて確認してください。

## 全体の流れ

1. 予知夢を見る
2. ゲームオーバー画面を表示する
3. ゲーム説明画面を表示する
4. 対策フェーズに入る
5. ショップに行くかどうかを選ぶ
6. ショップで対策グッズを購入する
7. 家の画面に戻る
8. 道具を持っている場合、家具を止める選択肢を出す
9. 緊急地震速報が鳴る
10. 災害当日を迎える
11. 対策できていればクリア、できていなければゲームオーバー

## ストーリー概要

主人公は予知夢で、自分が災害によって死亡する未来を見る。

今回の最小版では、家具が落ちてきて死亡する設定にする。ただし、予知夢の時点では、主人公はなぜ死んだのか具体的な原因を理解していない。

予知夢のあと、災害が数日後に起こることを知った主人公は、死を回避するために防災対策を行う。

今回は、最小構成として「家具止めを買って使ったかどうか」で生死が分岐する。

## 画面ごとの内容

### 1. 地震到来予知夢画面

目的:

- 予知夢として、未来のゲームオーバーを先に見せる
- プレイヤーに危機感を与える
- ただし、死因の具体的な理由はまだ明かさない

内容:

- 家具が落ちてきて主人公が死亡する
- 画面上はゲームオーバーに近い演出にする
- 主人公は「なぜ死んだのか」まではわからない

遷移:

- 次に `ゲームオーバー画面` へ進む

### 2. ゲームオーバー画面

目的:

- 予知夢の中で死亡したことを印象づける
- このままだと同じ未来になることを示す

内容:

- ゲームオーバー表示
- ただし、これは現実ではなく予知夢として扱う

遷移:

- 次に `説明画面` へ進む

### 3. 説明画面

目的:

- 主人公の予知夢について説明する
- ゲームの基本目的をプレイヤーに伝える

内容:

- 主人公は災害によって死ぬ未来を見た
- その未来を回避するために対策を行う必要がある
- 第三者視点で、ゲームの流れを簡単に伝える

遷移:

- 次に `スタート画面` へ進む

### 4. スタート画面

目的:

- 主人公が予知夢で死んだことを認識する
- 災害まで残り時間が少ないことを示す
- 対策フェーズへ入る動機づけをする

内容:

- あと数日で災害が起こる
- 何か対策をしないと死んでしまう
- 主人公が「対策しないと」と考える

遷移:

- 次に `選択画面` へ進む

### 5. 選択画面

目的:

- 最初の行動選択をさせる
- ショップに行くかどうかで大きく分岐させる

内容:

- 家に対策グッズがない
- 対策グッズを買いに行くかどうかを選ぶ

選択肢:

- 選択肢 1: 買いに行かない
- 選択肢 2: ショップへ行く

分岐:

- 選択肢 1: `選択肢1画面` へ
- 選択肢 2: `選択肢2画面（ショップ）` へ

## 分岐 2: ショップに行く

### 6-2. 選択肢2画面（ショップ）

目的:

- ショップで対策グッズを購入できるようにする
- 家具止めを買ったかどうかで分岐させる

内容:

- ショップで防災グッズを購入できる

## 当日イベント

災害当日は、緊急地震速報をきっかけに始まります。

最低限入れたい流れ:

1. 緊急地震速報が鳴る
2. 地震発生演出を入れる
3. 家具を止めていたかどうかを判定する
4. 家具止めありなら死亡回避
5. 家具止めなしなら予知夢通り死亡

## 最小版の判定ロジック

最小版では、次の条件で分岐します。

```ts
type StoryFlags = {
  visitedShop: boolean
  boughtFurnitureStopper: boolean
  usedFurnitureStopper: boolean
}
```

判定イメージ:

```ts
if (!visitedShop) {
  ending = 'bad_no_shop'
} else if (!boughtFurnitureStopper) {
  ending = 'bad_no_furniture_stopper'
} else if (!usedFurnitureStopper) {
  ending = 'bad_not_used_furniture_stopper'
} else {
  ending = 'clear'
}
```

## 実装時の注意

- 予知夢のゲームオーバーと、実際のゲームオーバーは意味が違うため、内部状態で区別する
- 最初の予知夢では、死因をはっきり説明しすぎない
- クリア後は、現実のプレイヤーに「自分の部屋はどうか」を問いかける演出を検討する
- ショップで家具止め以外を買った場合でも、今回は家具止めなしルートへ進む
- 将来的にアイテムが増えた場合、ブラフアイテムや別の対策ルートを追加できるようにする
